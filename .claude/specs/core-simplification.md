## Priority 1 — Correctness & dead code

### ✅ 1.1 `output/tables.py`: `convert_bool_to_emoji` is always-True

`bool(value)` on line 37 unconditionally casts the input to `bool` before
`is_bool(value)` is called, so the `is_bool` branch is always taken and the
final `return value` is genuinely dead code (`# pragma: no cover` confirms it).
`is_bool()` is not used anywhere else in the codebase.

**Fix**: Remove `is_bool()` and the dead branch from `convert_bool_to_emoji`.

```python
# Before
def convert_bool_to_emoji(value: object) -> str | bool | object:
    value = bool(value)
    if is_bool(value):
        tick = "[green]:heavy_check_mark:[/]"
        cross = "[red]:heavy_multiplication_x:[/]"
        return tick if value else cross
    return value  # pragma: no cover

# After
def convert_bool_to_emoji(value: object) -> str:
    """Convert a value to an emoji checkmark or cross."""
    return "[green]:heavy_check_mark:[/]" if value else "[red]:heavy_multiplication_x:[/]"
```

Remove `is_bool()` entirely and update its test if one exists.

### ✅ 1.2 `output/tables.py`: `CliTable.table` method shadows the instance attribute

`__init__` assigns `self.table = Table(...)` (a `rich.Table` instance).  The
class also defines a `table(self)` method that returns `self.table`.  After
`__init__`, calling `instance.table()` raises `TypeError` because `self.table`
is the `rich.Table` object, not the method.  The `# pragma: no cover` confirms
the method is never executed.

**Fix**: Remove the unreachable `table()` method — consumers already access the
attribute directly via `instance.table`.

## Priority 3 — Duplication & clarity

### ✅ 3.1 `decorators.py`: "walk to root context" pattern duplicated in `returns_response`

The same four-line pattern appears **twice** inside the `wrapper` function
(lines 337–341 for exception handling; lines 361–366 for dispatch):

```python
ctx = click_extra.get_current_context(silent=True)
root = ctx
if root is not None:
    while (parent := root.parent) is not None:
        root = parent
```

**Fix**: Extract a `_get_root_context` module-level helper:

```python
def _get_root_context() -> click_extra.Context | None:
    """Return the root Click context by walking up the parent chain."""
    ctx = click_extra.get_current_context(silent=True)
    if ctx is None:
        return None
    while ctx.parent is not None:
        ctx = ctx.parent
    return ctx
```

Replace both occurrences in `returns_response`.

### ✅ 3.2 `decorators.py`: `app_group` / `group` duplicate kwargs-splitting logic

Both functions repeat:

```python
config_fields = {f.name for f in fields(GroupConfig)}
config_kwargs = {k: v for k, v in kwargs.items() if k in config_fields}
click_kwargs = {k: v for k, v in kwargs.items() if k not in config_fields}
```

**Fix**: Extract a module-level helper:

```python
def _split_group_kwargs(kwargs: dict[str, Any]) -> tuple[dict[str, Any], dict[str, Any]]:
    """Split kwargs into GroupConfig fields and Click pass-through arguments."""
    config_fields = {f.name for f in fields(GroupConfig)}
    return (
        {k: v for k, v in kwargs.items() if k in config_fields},
        {k: v for k, v in kwargs.items() if k not in config_fields},
    )
```
